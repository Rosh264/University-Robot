2. Component Details by Category

2.1 Controller (1 item)

ComponentSpecificationFunction
ESP32 WROOM-32 Dual-core 240MHz, 520KB SRAM, WiFi 802.11 b/g/n, Bluetooth 4.2, 34 GPIO Central controller running audio capture, I2S playback, OLED display, servo control, WiFi communication to AI backend

2.2 Audio Input — INMP441 MEMS Microphone (1 item)

ComponentSpecificationFunction
INMP441 I2S digital output, 24-bit, SNR 61dB, sensitivity -26dBFS, VDD 1.8–3.3V Captures visitor voice questions as digital audio via I2S protocol (Port 0). Records 5 seconds at 16kHz 16-bit mono (~160KB)
Important: The INMP441 runs on 3.3V only. Do NOT connect to 5V. The L/R pin is tied to GND to select the left channel.

2.3 Audio Output — MAX98357A + Speaker (2 items)

ComponentSpecificationFunction
MAX98357A Amplifier I2S input, Class D amplifier, 3.2W output, VIN 2.5–5.5V Receives digital audio via I2S (Port 1) and amplifies it to drive the speaker. Plays greeting and AI response audio.
8Ω 3W Speaker 8 ohm impedance, 3 watt Connected to MAX98357A SPK+ and SPK– outputs. Produces audible speech for visitors.

2.4 Display — 1.3" OLED (1 item)

ComponentSpecificationFunction
1.3" OLED (SH1106) 128×64 pixels, I2C interface, address 0x3C, VCC 3.3V Displays welcome message, recording status, AI response text, and system status. Uses U8g2 library with software I2C on custom pins (SDA=GPIO21, SCL=GPIO19).

2.5 Sensor — PIR Motion (1 item)

ComponentSpecificationFunction
HC-SR501 PIR Sensor Detection range 3–7m, adjustable delay 0.3–200s, VCC 5–20V Detects approaching visitors via infrared motion. Triggers the welcome sequence when motion is detected. 10-second cooldown between triggers.

2.6 Actuator — SG90 Servo (1 item)

ComponentSpecificationFunction
SG90 Micro Servo Rotation 0–180°, operating voltage 4.8–6V, stall torque 1.8 kg·cm, PWM 50Hz Performs a waving greeting animation (3 waves: 30°→150°→90°) when a visitor is detected. Connected to GPIO18 via ESP32Servo library.

2.7 Power Supply (1 item)

ComponentSpecificationFunction
5V / 2A DC Supply Output: 5V regulated, 2A minimum capacity Powers ESP32 (via 5V pin or USB), SG90 servo, and MAX98357A amplifier. All grounds connected together. INMP441 and OLED use 3.3V from ESP32 regulator.

6. System Workflow

Step 1: System Initialization (Power On)

On power-up, the ESP32 initializes all modules in sequence: OLED display, PIR sensor, servo motor (centered at 90°), WiFi connection, I2S microphone (Port 0), and I2S speaker (Port 1). A startup screen is shown on the OLED with WiFi connection progress. Once connected, the system enters IDLE state.

Components: ESP32, OLED, PIR, Servo, INMP441, MAX98357A, WiFi

Step 2: Visitor Detection (PIR Motion Sensor)

The PIR sensor on GPIO27 continuously monitors for infrared motion. When a person approaches within 3–7 meters, the sensor output goes HIGH. A 10-second cooldown prevents multiple triggers from the same visitor. Upon detection, the system transitions from IDLE to WELCOME state.

Components: PIR HC-SR501 (GPIO27)

Step 3: Welcome Display (OLED)

The 1.3" OLED displays a welcoming screen: "Welcome to Garden City University" with decorative borders. The message is shown for 2 seconds before proceeding to the greeting phase.

Components: OLED SH1106 (GPIO21 SDA, GPIO19 SCL)

Step 4: Servo Greeting Animation

The SG90 servo motor performs a waving animation — three sweeps between 30° and 150°, then returns to center (90°). This simulates a hand-wave or head-nod greeting. Each wave takes approximately 600ms, totaling ~2 seconds for the full animation.

Components: SG90 Servo (GPIO18, PWM)

Step 5: Audio Greeting (Speaker)

The MAX98357A amplifier plays the greeting message "Hello! How can I help you?" through the speaker. If a pre-recorded WAV file exists in ESP32 SPIFFS, it plays that file. Otherwise, a tone sequence is played as a placeholder. The OLED shows "Speaking: Hello! How can I help you?".

Components: MAX98357A (GPIO25 LRC, GPIO26 BCLK, GPIO22 DIN), Speaker

Step 6: Voice Recording (Microphone)

The INMP441 MEMS microphone records the visitor's question for 5 seconds at 16kHz, 16-bit mono. Audio data is captured via I2S Port 0 and stored in a RAM buffer (~160KB). A progress bar is shown on the OLED. A WAV header is prepended to the raw PCM data.

Components: INMP441 (GPIO33 WS, GPIO32 SCK, GPIO35 SD)

Step 7: Audio Transmission (WiFi HTTP POST)

The recorded WAV audio is sent via HTTP POST to the backend server at the configured URL. The Content-Type is set to application/octet-stream. The total payload is approximately 160KB + 44 bytes (WAV header). Timeout is set to 30 seconds to allow for AI processing. The OLED shows "Processing..." with animation.

Components: ESP32 WiFi, HTTPClient

Step 8: AI Processing (Backend Server)

The Flask server receives the audio and performs three operations:

8a. Speech-to-Text: Uploads audio to Google Gemini API for transcription using the multimodal model (gemini-2.0-flash).

8b. AI Query Processing: Sends the transcribed text to Gemini with a university-specific system prompt containing information about departments, admissions, facilities, events, etc.

8c. Text-to-Speech: Converts the AI response text to speech audio using Gemini TTS (gemini-2.5-flash-preview-tts), outputting 16kHz 16-bit mono WAV for ESP32 playback.

The server returns a JSON response: { text, audio (base64 WAV), transcription, processing_time }.

Components: Flask Server, Google Gemini API (STT + AI + TTS)

Step 9: Response Display and Playback

The ESP32 receives the JSON response and performs two actions simultaneously:

OLED Display: The text response is word-wrapped and displayed on the 128×64 OLED. If the text exceeds 4 lines, it scrolls to a second screen after 3 seconds.

Speaker Playback: The base64-encoded audio is decoded back to WAV, and the PCM data is streamed through I2S Port 1 to the MAX98357A amplifier and speaker.

Components: OLED (I2C), MAX98357A (I2S), Speaker

Step 10: Follow-up Check

After the response, the system displays "Any other questions?" and listens for 2 seconds using a simple energy-based speech detection algorithm. If speech is detected (average energy above threshold), the system loops back to Step 6 (recording). If no speech is detected, it proceeds to goodbye.

Components: INMP441 (energy detection), OLED

Step 11: Goodbye and Return to Idle

The OLED displays "Thank You! Visit us again — Garden City University" with decorative borders. The servo performs one final wave animation. After 3 seconds, the system returns to IDLE state, displaying the ready screen and waiting for the next visitor.

Components: OLED, Servo, All modules return to standby

9. Quick Setup Instructions

Step 1: Hardware Assembly

Connect all components to the ESP32 according to the pin mapping in Section 3 and wiring diagram in Section 4. Ensure all grounds are connected together. Power the circuit with a 5V/2A supply.

Step 2: Arduino IDE Setup

Install Arduino IDE 2.x. Add ESP32 board support: File → Preferences → Additional Board Manager URLs → add https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json. Install "ESP32 by Espressif Systems" from Board Manager. Install libraries: U8g2, ESP32Servo, ArduinoJson from Library Manager.

Step 3: Configure Firmware

Open university_ai_bot.ino. Edit the WiFi credentials (WIFI_SSID, WIFI_PASSWORD) and the backend server URL (SERVER_URL — your computer's local IP, e.g., http://192.168.1.100:5000/api/query). Select board "ESP32 Dev Module" and upload.

Step 4: Backend Server Setup

Install Python 3.9+. Navigate to the backend/ folder. Copy .env.example to .env and add your Gemini API key (get free key at https://aistudio.google.com/apikey). Run: pip install -r requirements.txt, then python server.py.

Step 5: Test the System

Open the dashboard at http://localhost:5000 in a browser. Power on the ESP32. Verify "Bot Online" status appears on the dashboard. Wave your hand in front of the PIR sensor. The OLED should show the welcome message, the servo should wave, and the speaker should play the greeting. Speak a question and verify the AI response cycle completes.
